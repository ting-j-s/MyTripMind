## 1. 软件开发任务的描述

### 1.1 项目背景

随着旅游业的快速发展和个性化需求的增长，传统的旅游系统已无法满足用户对于智能化、个性化旅游体验的需求。本系统旨在为用户提供一个集旅游目的地推荐、路线规划、场所查询、旅游日记分享和美食推荐于一体的综合性旅游管理平台。

### 1.2 项目目标

1. **旅游推荐**：基于热度、评分和个人兴趣推荐景点，支持Top-K高效排序和模糊搜索
2. **路线规划**：实现单源最短路径（Dijkstra）和多景点闭环路径（TSP），支持多种策略
3. **场所查询**：基于道路网络的最近设施查询，不使用直线距离
4. **日记管理**：支持日记的创建、浏览、评分和全文检索
5. **美食推荐**：按热度、评分、距离排序，支持模糊搜索
6. **数据压缩**：使用霍夫曼编码压缩存储日记内容

### 1.3 技术约束

- **核心算法必须自己实现**：禁止直接调用数据库的排序、查找功能
- **必须使用自己设计的数据结构**：堆、哈希表、链表、图等
- **性能要求**：Top-10排序时间复杂度O(n log k)，Dijkstra使用堆优化

        """添加节点"""

        """添加边（道路）"""

        """获取邻居节点"""

## 5. 各模块设计说明

### 5.1 排序算法模块 (sort.py)

#### 5.1.1 快速排序

```
算法思想：分治法
1. 选择基准元素（pivot）
2. 分区：小于pivot的放左边，大于pivot的放右边
3. 递归排序左右两部分

时间复杂度：O(n log n)
空间复杂度：O(log n)（递归栈）
稳定性：不稳定
```

#### 5.1.2 堆排序

```
算法思想：利用堆这种完全二叉树结构
1. 构建最大堆
2. 依次取出堆顶（最大值），放到有序区

时间复杂度：O(n log n)
空间复杂度：O(1)
稳定性：不稳定
```

#### 5.1.3 Top-K部分排序

```
算法思想：维护一个大小为k的小顶堆
1. 遍历数据，将前k个元素入堆
2. 对于后续元素，如果比堆顶大，则替换堆顶
3. 最终堆中即为最大的k个元素

时间复杂度：O(n log k)
空间复杂度：O(k)
适用场景：数据量大，只取前10名
```

### 5.2 最短路径算法模块 (dijkstra.py)

#### 5.2.1 标准Dijkstra

```
算法思想：贪心 + 优先队列
1. 初始化：起点距离为0，其他为无穷大
2. 用最小堆维护待处理节点
3. 取出堆顶节点（当前最短距离），更新其邻居距离
4. 重复直到目标节点被取出或堆空

时间复杂度：O((V+E) log V)
空间复杂度：O(V)
```

#### 5.2.2 时间最短Dijkstra变体

```
权重计算：time = distance / (congestion × ideal_speed)
拥挤度(congestion)：0-1之间，越大越堵
真实速度 = congestion × ideal_speed

适用于：考虑道路拥挤情况下的最短时间路径
```

#### 5.2.3 交通工具约束Dijkstra

```
边过滤：在扩展邻居时，根据交通方式过滤道路
- 步行：所有道路
- 自行车：仅road_types包含"自行车"的道路
- 电瓶车：仅road_types包含"电瓶车"的道路

适用于：不同交通方式只能走允许的道路
```

### 5.3 TSP算法模块 (tsp.py)

#### 5.3.1 贪心最近邻

```
算法思想：
1. 从起点开始
2. 选择距离当前位置最近的未访问节点
3. 重复直到访问完所有节点
4. 最后返回起点形成闭环

时间复杂度：O(n²)
优点：简单快速
缺点：可能陷入局部最优
```

#### 5.3.2 2-opt优化

```
算法思想：迭代改进
1. 给定一个贪心解的路径
2. 尝试交换路径中的两条边
3. 如果交换后路径变短，则接受
4. 重复直到没有改进

时间复杂度：O(n² × iterations)
优点：显著改善解的质量
```

### 5.4 哈希表模块 (hash_table.py)

```
数据结构：链地址法哈希表
哈希函数：除法取余法
冲突解决：链地址法（拉链法）

核心操作：
- insert(key, value)：O(1)平均
- get(key)：O(1)平均
- delete(key)：O(1)平均
- resize()：动态扩容，保持负载因子

用途：用户ID快速查找、日记ID快速查找
```

### 5.5 模糊搜索模块 (fuzzy_search.py)

```
算法思想：编辑距离（Levenshtein Distance）
编辑操作：插入、删除、替换
编辑距离：将字符串A变成字符串B所需的最少编辑次数

优化：
- 设置最大编辑距离阈值
- 超过阈值的直接排除
- 使用缓存避免重复计算

时间复杂度：O(n × m × k)，n和m为字符串长度，k为阈值
```

### 5.6 全文搜索模块 (text_search.py)

```
算法思想：倒排索引
1. 构建索引：word -> [doc_id, ...]
2. 查询时：对查询词取并集
3. 返回匹配文档列表

优化：
- 停用词过滤（的、了、在等）
- 统一小写
- 支持多词查询

时间复杂度：O(1) 查找 + O(n) 返回结果
```

### 5.7 霍夫曼压缩模块 (compression.py)

```
算法思想：变长编码，高频字符用短码
1. 统计字符频率
2. 构建霍夫曼树（最小堆）
3. 生成编码表（左0右1）
4. 编码原文
5. 存储：编码表 + 编码内容

压缩率：通常30%-50%

用途：日记内容压缩存储
```

### 5.8 图结构模块 (graph.py)

```
存储方式：邻接表
节点：建筑物、设施、景点
边：道路

数据结构：
adj_list[node_id] = [(neighbor_id, distance, attrs), ...]

核心操作：
- add_node(node_id, data)
- add_edge(from_node, to_node, **attrs)
- get_neighbors(node_id)
- dijkstra(start, end, weight='distance', transport=None)

适用于：道路网络最短路径计算
```
